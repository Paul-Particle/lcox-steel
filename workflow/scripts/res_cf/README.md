# RES capacity factors (Atlite-based)

This folder contains **hourly, per-unit (0–1) renewable energy capacity factor (CF) time series**
derived using **Atlite + ERA5** for use in energy system and techno-economic models
(e.g. LCOX, electrolyser optimisation).

The pipeline has been validated and executed for the following countries (year 2023):

- Germany (DE)  
- France (FR)  
- Spain (ES)  
- Australia (AUS)  
- Brazil (BRA)

All countries use identical methodological and technology assumptions to ensure
cross-country comparability.

---

# 🔴 IMPORTANT UPDATE (compared to previous version)

The “best-site” methodology has been **fundamentally revised**.

### Previously:
- Best-site CFs were generated by **scaling national CFs with uplift factors (P90/P95)**
- This preserved temporal structure but **did not represent a real site**

### Now:
- Best-site CFs are derived from **actual grid-cell time series**
- Selected directly from Atlite CF fields
- Include **wind power-curve smoothing + spatial aggregation**

👉 This makes best-site CFs:
- physically interpretable  
- suitable for project-level modelling  
- directly comparable to national CFs

# Outputs (authoritative)

## National-average CFs

- `de_cf_2023.parquet`
- `fr_cf_2023.parquet`
- `es_cf_2023.parquet`
- `aus_cf_2023.parquet`
- `bra_cf_2023.parquet`

Each file contains:

- `time` (hourly, full calendar year)
- `wind_onshore_cf`
- `solar_cf`

Each file contains **8760 hourly values** (non-leap year) and is intended for
downstream modelling.

### Definition
**National-average CFs**:  
Annual hourly CF time series aggregated at country level from Atlite cutouts using the regional **indicator matrix**.

---

## Best-site P95 CFs

- `resources/res_cf/<cc>_cf_2023_bestsite_p95.parquet`

### Definition
**Best-site P95 CFs**:  
Annual hourly CF time series for a representative high-quality site selected from the country CF grid at the **95th percentile of annual mean capacity factor**.

### Method (final implementation)

Best-site wind CF is represented as:

> “The hourly output of the P95 ERA5 grid cell using Atlite’s built-in smoothed wind power curve, which better approximates farm-level production than an unsmoothed single-turbine point response.”

Selection:
- Grid cells ranked by **annual mean CF**
- P95 threshold computed
- Representative cell selected (closest to threshold)

Wind-specific post-processing:

> “Best-site wind profiles are derived from the P95 ERA5 grid cell and spatially averaged over a 3×3 cell cluster to approximate wind farm-scale variability and avoid artificial saturation at rated power.”

👉 Final method uses **mean-based P95 selection** (median-based selection was tested and rejected).

# Method summary

1. **Climate data**
   - ERA5 reanalysis via Atlite
   - Time chunking for numerical/CDS stability:
     - DE/FR/ES/AUS/BRA: quarterly cutouts (Q1–Q4)
   - Large-country grid coarsening for CDS job stability and runtime:
     - AUS/BRA cutouts use `dx=0.5`, `dy=0.5`
     - EU countries use default resolution (no `dx/dy` override)

2. **Spatial aggregation**
   - Country geometry from `regions.parquet`
   - Area-weighted aggregation using Atlite **indicator matrix**
   - Resulting CFs represent **national, climate-driven availability**

3. **Technology assumptions**
   - **Onshore wind:** single representative turbine  
     (`Vestas_V112_3MW`)
   - **Solar PV:** crystalline silicon (`CSi`), latitude-optimal tilt
   - Same technology assumptions used across countries for comparability

4. **Temporal handling**
   - CFs computed hourly per cutout segment (Q1–Q4)
   - Pure concatenation to full year (no temporal averaging)
   - Aggregation (monthly/annual) performed only for QC or reporting

# Interpretation note

Capacity factors represent **country-level, area-weighted technical availability**.
They are suitable for cross-country comparison and system-level modelling.

Project-level LCOX at high-resource sites (e.g., strong wind or solar corridors)
may yield materially higher CFs than these national averages.

Best-site CFs represent **resource-optimised project locations** and should be interpreted as such.

# Scripts (pipeline overview)

- `scripts/res_cf/build_regions.py`  
  Builds `resources/shapes/regions.parquet` from Natural Earth country geometries.

- `scripts/res_cf/build_offshore_regions.py`  
  Builds `resources/shapes/offshore_regions.parquet` from EEZ polygons restricted to a near-shore deployment band.

- `scripts/res_cf/download_cutout.py`  
  Creates ERA5 cutouts (`cutouts/*.nc`) for a given country and time segment
  (Q1–Q4). For large domains (AUS/BRA), uses coarser grid (`dx=0.5`, `dy=0.5`)
  to avoid CDS job failures and reduce runtime/memory.

- `scripts/res_cf/build_cf_timeseries.py`  
  Computes per-unit hourly CF time series (wind onshore + solar) from a single cutout
  using indicator-matrix aggregation over the country geometry. Outputs per-segment CSVs
  (e.g. `*_q1.parquet`).

- `scripts/res_cf/concat_quarters.py`  
  Concatenates quarterly outputs (Q1–Q4) into full-year per-technology series:
  `*_wind_onshore_cf_2023.parquet`, `*_solar_cf_2023.parquet`, `*_wind_offshore_cf_2023.parquet` (8760 hours).

- `scripts/res_cf/combine_techs.py`  
  Merges yearly wind + solar into the final modelling inputs:
  `*_cf_2023.parquet` with columns `time`, `wind_onshore_cf`, `wind_offshore_cf`, `solar_cf`.

- `scripts/res_cf/determine_resource_spread.py`  
  Computes intra-country spatial resource distribution metrics from the existing Atlite cutouts.  
  For each country and technology:
  - derives annual mean CF per grid cell (no national aggregation)
  - calculates area-weighted spatial statistics (mean, P90, P95, max)
  - computes uplift factors relative to the national mean  
  Outputs:
  - `resources/res_cf/resource_spread_2023.parquet`

- `scripts/res_cf/determine_bestsite_p95.py`  

  ### Updated behaviour:
  Generates “best-site” hourly CF time series by **direct extraction from Atlite CF grids**.

  - builds full CF grid (`cf_year`)
  - applies wind smoothing (`smooth=True`)
  - selects P95 grid cell (mean-based)
  - extracts hourly CF time series
  - applies 3×3 spatial averaging (wind only)

  Outputs:
  - `resources/res_cf/<cc>_cf_2023_bestsite_p95.parquet`

  No longer:
  - multiplies national CF by uplift factors
  - uses resource_spread outputs for time series generation

- `scripts/res_cf/determine_complementarity.py`  
  Screens all valid (onshore, offshore, solar) grid-cell triplets per country and ranks them
  by a complementarity score (coincidence - correlation). Outputs top-N triplets and national
  mean profiles for use in PyPSA scenarios.

- `scripts/res_cf/diag_qc_yearly_cf.py`  
  Structural + physical sanity checks on yearly CF outputs.

# Offshore wind spatial baseline (EEZ source)

For the offshore wind extension, offshore areas are derived from **Exclusive Economic Zone (EEZ)** polygons and then restricted to a **near-shore deployment band**.

**Source dataset**  
Marine Regions – **World EEZ v12 (2023-10-25)**  
Website: https://www.marineregions.org/downloads.php

Downloaded package used:
- `World EEZ v12` → `Shapefile`

Stored locally as:
- `data/shapes/offshore_zones/eez_v12.shp`

### Baseline filtering logic

The EEZ dataset is filtered to the project countries:

- `DEU`
- `FRA`
- `ESP`
- `AUS`
- `BRA`

and restricted to:

- `POL_TYPE == "200NM"`

This keeps only the clean national EEZ polygons and excludes **joint regime** areas.

### Mapping to model regions

- `DEU -> DE`
- `FRA -> FR`
- `ESP -> ES`
- `AUS -> AUS`
- `BRA -> BRA`

### Offshore geometry construction

offshore area = (EEZ ∩ near-shore buffer) − land

# Spatial resource spread (context)

The resource spread analysis remains part of the pipeline for **diagnostics and reporting**, but is no longer used to generate best-site time series.

It quantifies intra-country heterogeneity and potential uplift but does not define temporal profiles anymore.
